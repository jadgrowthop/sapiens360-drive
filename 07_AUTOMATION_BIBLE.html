Section 2

ManyChat — 5 Flows

Flow 1 — Commentaire "SAPIENS"

90% du flux
+

Trigger : Commentaire contenant SAPIENS / sapiens / PEAU / peau

  1. Public reply auto (reponse sous le post)
  2. DM capture email
  3. Webhook GHL (creation/update contact)
  4. DM lien VSL
  5. Timer H+3 — Jad envoie vocal personnalise
  6. Timer H+24 — DM relance
  7. Timer H+72 — DM final
  8. STOP

Tags appliques : email-capture vsl-envoyee vocal-a-envoyer lead-froid-manychat

Flow 2 — Nouveau Follow

+

Trigger : Nouveau follower @geo_le_sapiens

  1. Delai 5 minutes
  2. DM : "C'est quoi qui t'a amene ?"
  3. Routing A/B/C/D selon reponse
  4. Si probleme de peau → redirection vers Flow 1

Flow 3 — DM Entrant Spontane

+

Trigger : DM entrant (incoming message)

Keywords detectes : rosacee, acne, eczema, psoriasis, peau, boutons, plaques, rougeurs, inflammation, dermato

  1. Si keyword detecte → qualification → Flow 1
  2. Si aucun keyword → tag dm-non-qualifie → Jad traite manuellement

Flow 4 — Relance Post-VSL

+

Trigger : Tag vsl-envoyee + 48h sans tag rdv-reserve

  1. DM relance + lien Calendly
  2. +72h si rien → tag lead-froid-post-vsl
  3. STOP → GHL nurture prend le relais

Flow 5 — Quiz en Bio

+

Trigger : Click sur le lien quiz dans la bio Instagram

  1. Q1 : Condition (psoriasis, acne, rosacee, eczema, digestion)
  2. Q2 : Duree du probleme
  3. Q3 : Tentatives passees
  4. Q4 : Impact sur la vie (champ libre — gold pour closing)
  5. Q5 : Pret au changement
  6. Capture email
  7. Resultat personnalise
  8. VSL + lien Calendly
  9. Webhook GHL avec tous les champs

Regle Anti-Overlap

Chaque workflow retire le contact des workflows precedents pour empecher les envois en double. Un contact ne peut etre que dans UN workflow a la fois.

WF1 — VSL Relance

+

Objectif : Relancer les leads qui ont recu la VSL mais n'ont pas encore reserve de call.

  1. Email J+1 : Rappel VSL + lien Calendly
  2. Email J+3 : Temoignage + urgence
  3. Email J+5 : Derniere chance
  4. Si aucune action → tag lead-froid-post-vsl

WF2 — Pre-Close Booking

+

Objectif : Preparer le prospect au closing call. Maximiser le show rate.

  1. Email immediat : confirmation + consignes pre-call
  2. Email J-1 : rappel + temoignage condition similaire
  3. SMS J-0 H-2 : rappel final

WF3/3b — Post-Call Rescue

+

Objectif : Recuperer les prospects qui n'ont pas close pendant le call.

WF3 = calls de Jad | WF3b = calls de Geoffrey

  1. Email H+2 : recap valeur + reponse objection principale
  2. Email J+1 : temoignage + urgence
  3. Email J+3 : offre limitee / dernier push
  4. Si rien → tag lead-froid-post-call

WF4 — No-Show Recovery

+

Objectif : Recuperer les no-shows et les remettre en booking.

  1. Email immediat : "On t'attendait" + nouveau lien Calendly
  2. Email J+1 : rappel + social proof
  3. Tag no-show applique automatiquement

WF6 — Onboarding

+

Objectif : Accueillir le nouveau client et l'amener au premier resultat rapide.

  1. Email immediat : bienvenue + acces Whop + consignes J1
  2. Email J+1 : premiere action a faire
  3. Email J+3 : check-in
  4. Email J+7 : premier milestone

WF9 — Testimonial Collection

+

Objectif : Collecter des temoignages a des moments strategiques.

  1. J+14 : premier check-in + demande feedback
  2. J+30 : demande temoignage video
  3. J+60 : relance temoignage si pas fait
  4. J+90 : demande photo avant/apres

Section 7

Troubleshooting

ManyChat ne repond pas

Le flow ne se declenche pas quand un utilisateur commente ou envoie un DM.

Verifier : flow actif • triggers configures • connexion Instagram valide • permissions de page

GHL emails non envoyes

Le workflow est sense envoyer un email mais le contact ne recoit rien.

Verifier : workflow actif • tag correct sur le contact • deliverabilite du domaine • email non en spam

Calendly webhook en echec

Les bookings ne remontent pas dans GHL.

Verifier : URL du webhook • test event dans Calendly • logs Netlify function • mapping des champs

Whop webhook en echec

Les paiements ne sont pas detectes dans GHL.

Verifier : Netlify function active • logs dans Netlify • format du body Whop • email match

Tags non appliques

Un contact devrait avoir un tag mais il manque.

Verifier : conditions du workflow • regles anti-overlap • le contact est-il dans le bon workflow ?

Custom fields vides

Les champs personnalises ne sont pas remplis.

Verifier : cron backfill actif • mapping des noms (camelCase vs snake_case) • webhook body format

Emails en spam

Les emails de GHL arrivent dans les spams du prospect.

Verifier : DKIM configure • SPF valide • DMARC en place • domaine d'envoi warmup • contenu non spammy

Donnees formulaire manquantes

Le formulaire est soumis mais les donnees n'apparaissent pas dans GHL.

Verifier : statut du cron backfill • mapping des champs du form • version du formulaire (v3) • logs
